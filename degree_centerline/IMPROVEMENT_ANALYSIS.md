## 2. 问题 1：末端线段缺失

### 2.1 根因分析

中线未到达 polygon 末端，有三个叠加原因：

#### 原因 A：Voronoi 图的天然特性

Voronoi 顶点（中线节点）位于**两个或多个边界点等距**的位置。在 polygon 的尖角末端，等距点天然位于角尖**内部**，离尖端有一定距离。Voronoi 骨架本身不会产生"到达边界"的节点。

```
       polygon 尖角端

       ╲                ╱
        ╲              ╱
         ╲    ●——●    ╱     ← Voronoi 顶点在这里（远离尖端）
          ╲  ╱    ╲  ╱
           ╲╱      ╲╱       ← 尖端点在下方
            ▼      ▼         没有 Voronoi 顶点到达这里
```

**代码定位**：`_centerline_voronoi_fast()` 第 919 行 — `vor = Voronoi(pts)` 产生的骨架天然不到达边界。

#### 原因 B：生成距离过滤（`min_gen_dist` 过滤器）

```python
# centerline_degree.py 第 920, 939 行
min_gen_dist = 3.0 * actual_distance
gen_filter_pass = ~(same_ring & (gen_dist < min_gen_dist))
```

在尖角末端，**对面两侧的边界点**距离很近（角越尖距离越小），
导致它们的 Voronoi 脊线的 `gen_dist`（生成点距离）小于 `min_gen_dist`，
被**错误地过滤掉**——认为是噪声，实则是有意义的末端骨架。

**这是最主要的过滤原因**。

#### 原因 C：预剪枝阈值

```python
# centerline_degree.py 第 737 行
pre_prune_threshold = 3.0 * densify_distance
G = _prune_branches(G.copy(), pre_prune_threshold)
```

末端的最后一小段骨架（从最后一个分叉点到叶节点）如果长度 < `3 × densify_distance`，
会被 `_prune_branches` 作为"噪声短枝"直接删除。

叠加效果：**原因 B 移除了末端的脊线** → 本来就短的末端段变得更短 → **原因 C 进一步删除残余**。

---

### 2.2 解决方案

#### 方案 1-A：末端延伸——后处理投影法

**思路**：骨架提取完成后，对每个**叶节点**（degree=1），将其沿方向向外延伸，
投影到最近的 polygon 边界上。

**算法**：

```
对每个叶节点 L：
  1. 取 L 和它的唯一邻居 N，计算方向向量 d = L - N（归一化）
  2. 沿 d 方向延伸一条射线 R
  3. 计算 R 与 polygon 所有边界边的交点
  4. 取最近的交点 P
  5. 添加一条从 L 到 P 的新边
```

**处理方角**：当射线与方角边缘相交时，交点自然落在边缘上的垂直中点附近。

```
   延伸前：               延伸后：
   
    ╲   polygon  ╱         ╲   polygon  ╱
     ╲  ●——●    ╱           ╲  ●——●    ╱
      ╲╱    ╲  ╱             ╲╱    ╲  ╱
              ╲╱                     ╲╱
                                      ●  ← 新增的延伸端点（射线与边界的交点）
```

**优点**：
- ✅ 概念简单，实现直接
- ✅ 不影响现有骨架质量——纯后处理
- ✅ 尖角/方角均自然处理（射线投射到边界，结果取决于几何形态）
- ✅ 不改变 Voronoi 过滤逻辑，保持向量化性能

**缺点**：
- ⚠️ 方向延伸可能偏离真正的中线方向（如果叶节点附近的骨架有弯曲）
- ⚠️ 对环形 polygon 可能误判（叶节点不代表末端）
- ⚠️ 需要实现射线-多边形边界求交（增加代码复杂度 ~40 行）

**复杂度**：O(L × K)，L=叶节点数，K=环边数（通常很小）

**代码修改位置**：`_extract_branching_skeleton()` 末尾，Step 5 之后新增 Step 6

---

#### 方案 1-B：末端延伸——多点平滑方向投影法

**思路**：与方案 1-A 类似，但用叶节点附近的**多个节点**拟合方向，使投影更稳定。

**算法**：

```
对每个叶节点 L：
  1. 沿骨架从 L 向内走 k 个节点（如 k=3），收集路径 [L, N1, N2, N3]
  2. 用最小二乘拟合一条直线，得到末端方向向量 d
  3. 沿 d 方向从 L 投影到 polygon 边界
  4. 取最近的交点 P，添加 L→P 边
```

**优点**：
- ✅ 方向比单邻居法更稳定（抗骨架末端弯曲噪声）
- ✅ 不影响现有骨架质量

**缺点**：
- ⚠️ 实现略复杂（需要沿图行走 + 线性拟合）
- ⚠️ k 值的选择需要调参（太小无效，太大可能跨过分叉点）
- ⚠️ 对短骨架段（< k 个节点）需要退化处理

**复杂度**：O(L × (k + K))

**代码修改位置**：同方案 1-A

#### 方案 1-C：放宽末端过滤——降低 `min_gen_dist` 系数

**思路**：根因 B 的 `min_gen_dist = 3.0 × actual_distance` 过于激进。
在 polygon 尖角处，对面边界的 generator 距离本来就小，
降低系数可以让这些脊线通过过滤。

**具体修改**：

```python
# 原代码（第 920 行）：
min_gen_dist = 3.0 * actual_distance

# 方案 1-C：降低到 2.0 或 1.5
min_gen_dist = 2.0 * actual_distance  # 选项 a
min_gen_dist = 1.5 * actual_distance  # 选项 b
```

**优点**：
- ✅ 改动极小（改一个常量）
- ✅ 从根本上解决"末端脊线被过度过滤"的问题
- ✅ 无需额外后处理

**缺点**：
- ⚠️ **全局影响**——放宽过滤可能让其他位置的噪声脊线也通过，降低整体骨架质量
- ⚠️ 难以找到普适的最优值——不同 polygon 形状有不同的最佳阈值
- ⚠️ 不能单独控制"只放宽末端区域"

**复杂度**：O(1) — 仅改常量

**代码修改位置**：`_centerline_voronoi_fast()` 第 920 行

---

#### 方案 1-D：自适应 `min_gen_dist`——根据局部角度调整

**思路**：不用全局常量，而是为每条脊线**根据其局部几何特征**动态计算过滤阈值。
在尖角区域（generator 的局部角度小），放宽 `min_gen_dist`；
在宽阔区域（局部角度大），保持严格过滤。

**算法**：

```
对每条脊线 i 的两个 generator 点 G1, G2：
  1. 取 G1 在边界上的前后邻居，计算局部转折角 θ1
  2. 取 G2 的局部转折角 θ2
  3. θ = min(θ1, θ2)
  4. 如果 θ < 阈值（如 60°），说明在尖角附近，使用较小的 min_gen_dist
  5. 否则，使用正常的 min_gen_dist
```

**优点**：
- ✅ 精准定位尖角区域，不影响其他区域
- ✅ 自适应——无需调整全局参数

**缺点**：
- ⚠️ 实现复杂度高（需要计算每个 generator 点的局部角度）
- ⚠️ 性能开销——需要对每条脊线查找邻居并计算角度
- ⚠️ "转折角"的定义和阈值选择需要仔细设计

**复杂度**：O(R × 2)，R=脊线数

**代码修改位置**：`_centerline_voronoi_fast()` 第 933–941 行附近

---

#### 方案 1-E：降低预剪枝阈值

**思路**：根因 C 的 `pre_prune_threshold = 3.0 × densify_distance` 可能过于激进。
降低系数可以保留末端短段。

```python
# 原代码（第 737 行）：
pre_prune_threshold = 3.0 * densify_distance

# 方案 1-E：降低到 1.5 或 2.0
pre_prune_threshold = 1.5 * densify_distance
```

**优点**：
- ✅ 改动极小
- ✅ 保留更多末端细节

**缺点**：
- ⚠️ **与问题 2 冲突**——降低预剪枝会保留更多噪声短枝（加重钝角分支问题）
- ⚠️ 全局影响——非末端区域的噪声毛刺也会被保留
- ⚠️ 仅部分缓解问题（末端脊线本身可能在 Voronoi 阶段已被删除）

**复杂度**：O(1)

**代码修改位置**：`_extract_branching_skeleton()` 第 737 行

## 3. 问题 2：钝角拐角处多余分支

### 3.1 根因分析

在 polygon 的**钝角拐角**处（如 120°–170° 的轮廓弯角），Voronoi 图会在弯角处
产生一个**指向弯角顶点方向的短脊线分支**。

这是因为 Voronoi 图的数学性质决定的：弯角顶点附近，有一组边界点的等距面
向外突出形成一条短的"脊线臂"。

```
   polygon 轮廓走向：
   
        ╲                  ╱
         ╲                ╱
          ╲     ●════●   ╱    ← 主干中线
           ╲   ╱        ╱
            ╲ ╱        ╱
             ●        ╱
            ╱│╲      ╱         ← 钝角拐角处
           ╱ │ ╲    ╱
          ╱  ●  ╲  ╱            ← 多余的短分支（指向弯角）
         ╱       ╲╱
```

**当前过滤机制为何未能去除**：

1. **预剪枝**（`3 × densify_distance`）：如果分支长度刚好 > 阈值（取决于 polygon 宽度和弯角角度），就不会被移除。

2. **`min_branch_ratio` 过滤**（默认 10%）：如果分支长度 > 最长段的 10%，也不会被移除。
   对于较窄的 polygon，弯角处的分支可能占最长段的 15–25%，恰好超过阈值。

3. **没有角度感知**：当前算法**纯粹基于长度**做过滤，完全不考虑分支是否对应一个"有意义的形态分叉"还是仅仅是一个"轮廓弯角"。

**核心问题**：算法缺乏区分**真正的形态分叉**（T 型、Y 型）与**轮廓弯角**（拐弯处的弯曲）的能力。

---

### 3.2 解决方案

#### 方案 2-A：基于 polygon 轮廓角度的分支过滤

**思路**：钝角处的分支总是**指向 polygon 轮廓的弯角顶点**。
可以检测分支叶节点是否靠近一个**钝角轮廓顶点**，如果是，视为非结构性分支并删除。

**算法**：

```
对每个保留的 terminal 段（叶节点 L → 分叉点 J）：
  1. 找到 L 在 polygon 外环上的最近顶点 V
  2. 计算 V 的轮廓转折角 α（V 的前一个顶点、V、V 的下一个顶点组成的角）
  3. 如果 α > 阈值（如 150° 表示很钝的角）→ 删除该分支
```

**角度计算示例**：

```
       A                              A
       │                              │
       │                               ╲
       V────B    ← 180°（直角）          V────B    ← 150°（钝角）
```

```python
# 计算顶点 V 的转折角
vec1 = A - V  # V 的前一条边方向
vec2 = B - V  # V 的后一条边方向
cos_alpha = dot(vec1, vec2) / (|vec1| * |vec2|)
alpha = arccos(cos_alpha)  # 内角

# 如果 alpha > 150°（钝角阈值），认为是轮廓弯角而非结构分叉
```

**优点**：
- ✅ 直接利用 polygon 几何信息，精准定位钝角
- ✅ 只过滤钝角分支，保留尖角/T形分叉
- ✅ 参数直观（角度阈值，如 150°）

**缺点**：
- ⚠️ 需要访问原始 polygon 顶点数据（需要传递到 `_extract_branching_skeleton`）
- ⚠️ 密化后的边界点很多，需要先映射回原始顶点或对密化后的点计算角度
- ⚠️ 对圆滑弯角（无明显顶点）可能不够灵敏
- ⚠️ 增加代码复杂度和耦合度

**复杂度**：O(T × V_orig)，T=terminal 段数，V_orig=原始顶点数

**代码修改位置**：`_extract_branching_skeleton()` Step 4 内新增角度过滤逻辑

#### 方案 2-B：基于分支方向 vs 主干方向的角度过滤

**思路**：真正的形态分叉（T 形、Y 形）的分支方向与主干方向夹角通常 > 45°。
而钝角处的多余分支几乎与主干方向**平行或接近平行**（夹角 < 30°），
因为它只是主干在弯角处的"延伸毛刺"。

**算法**：

```
在分叉点 J，对于每个 terminal 段 S（J → L）：
  1. 计算分支方向 d_branch = L 的位置 - J 的位置（归一化）
  2. 计算 J 处主干方向 d_main = 主干的下一个节点 - J（归一化）
  3. 计算夹角 θ = arccos(|dot(d_branch, d_main)|)
  4. 如果 θ < 阈值（如 30°）→ 分支几乎与主干平行 → 删除
```

```
   真正的分叉（保留）：           钝角伪分支（删除）：
   
       ╱ d_branch                    d_main
      ╱   θ ≈ 60°                  ↗
     J─────→ d_main              J─────→ d_main
      ╲                           ╲ d_branch
       ╲                            ╲  θ ≈ 15° ← 几乎平行！
```

**优点**：
- ✅ 不需要访问原始 polygon 数据——纯粹基于骨架图的拓扑和几何
- ✅ 物理含义直观：平行于主干的分支不是真正的"叉"
- ✅ 实现相对简单（~25 行）

**缺点**：
- ⚠️ 需要在分叉点 J 正确识别"主干方向"（如果有多个分支，哪个是主干？）
- ⚠️ 阈值选择需要经验（太小不够过滤，太大误删真分支）
- ⚠️ 对弯曲主干（主干在 J 处就有大角度变化）可能误判

**复杂度**：O(T)

**代码修改位置**：`_extract_branching_skeleton()` Step 4 新增方向角度判断

---

#### 方案 2-C：提高 `min_branch_ratio` 默认值

**思路**：将 `min_branch_ratio` 从 0.1（10%）提高到 0.15 或 0.2（15–20%），
让更多的短分支被过滤。

```python
# 原代码（第 702 行默认参数）：
min_branch_ratio: float = 0.1

# 方案 2-C：提高默认值
min_branch_ratio: float = 0.15  # 或 0.20
```

**优点**：
- ✅ 改动极小（一个默认值）
- ✅ 能过滤掉更多短分支

**缺点**：
- ⚠️ **非选择性**——真正的短分叉（如 T 形小分支）也会被误删
- ⚠️ 阈值依赖 polygon 形状——不同形状需要不同的最佳值
- ⚠️ 无法区分"钝角伪分支"和"有意义的短分支"

**复杂度**：O(1)

**代码修改位置**：`_extract_branching_skeleton()` 函数签名第 702 行

---

#### 方案 2-D：增加预剪枝阈值

**思路**：提高 `pre_prune_threshold` 系数（从 3.0 增加到 5.0 或 6.0），
在拓扑分析之前就移除更多短枝。

```python
# 原代码（第 737 行）：
pre_prune_threshold = 3.0 * densify_distance

# 方案 2-D：提高系数
pre_prune_threshold = 5.0 * densify_distance
```

**优点**：
- ✅ 改动极小
- ✅ 在更早阶段清理噪声

**缺点**：
- ⚠️ **与问题 1 严重冲突**——更激进的预剪枝会使末端缺失更严重
- ⚠️ 非选择性——不区分钝角伪分支和有意义短分支
- ⚠️ 对不同 densify_distance 的效果不一致

**复杂度**：O(1)

**代码修改位置**：`_extract_branching_skeleton()` 第 737 行

---

#### 方案 2-E：基于 polygon 宽度比的分支过滤

**思路**：在分支叶节点处，测量 polygon 的**局部宽度**（叶节点到最近边界的距离 × 2）。
如果分支长度 < 局部宽度 × 系数（如 0.5），说明分支太短、不是真正的形态分叉。

**算法**：

```
对每个 terminal 段（L → J）：
  1. 计算 L 到最近 polygon 边界的距离 d_L
  2. 计算 J 到最近 polygon 边界的距离 d_J
  3. 局部宽度 W ≈ max(d_L, d_J) × 2
  4. 如果 段长度 < W × 0.5 → 删除（分支长度不足 polygon 宽度的一半）
```

**优点**：
- ✅ 自适应——自动适配不同宽度的 polygon 部位
- ✅ 物理含义清晰：分支至少要跨越半个 polygon 宽度才有意义

**缺点**：
- ⚠️ 需要计算点到边界的距离（需要访问原始 polygon 数据）
- ⚠️ 在 polygon 宽度变化剧烈的区域可能不准确
- ⚠️ 实现复杂度中等（~30 行）

**复杂度**：O(T × K)

**代码修改位置**：`_extract_branching_skeleton()` Step 4

#### 方案 2-F：组合过滤——长度比 + 角度 + 宽度

**思路**：综合多个维度进行评分，只有"得分足够低"的分支才被删除。

**评分公式（示例）**：

```
score = w1 × (段长度 / max_len)        # 长度因子
      + w2 × (分支角度 / 90°)          # 角度因子（与主干夹角越大越有意义）
      + w3 × (段长度 / 局部宽度)       # 宽度比因子

if score < threshold → 删除
```

**优点**：
- ✅ 最强的判别能力——多维度综合评估
- ✅ 权重可调

**缺点**：
- ⚠️ 参数多（w1, w2, w3, threshold），调参复杂
- ⚠️ 实现和维护成本高
- ⚠️ 对用户不透明——"为什么这个分支被删了"不直观

**复杂度**：O(T × K)

**代码修改位置**：全面改造 `_extract_branching_skeleton()` Step 4
